# E-commerce Platform Search Function

Big O notation describes how runtime grows as input size grows.

Search complexity:
- Linear search: best `O(1)`, average `O(n)`, worst `O(n)`.
- Binary search: best `O(1)`, average `O(log n)`, worst `O(log n)`, but the array must be sorted.

Binary search is better for large catalogs that are searched often. Linear search is acceptable for small or unsorted lists.
